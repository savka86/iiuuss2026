ИУС 2.0 — бесплатный вариант заново
Google Таблица + Apps Script + Google Drive

ФАЙЛЫ ДЛЯ GITHUB PAGES
1) index.html — страница ученика.
2) expert.html — страница учителя.
3) styles.css — оформление.
4) apps-script-config.js — ссылка на Apps Script.

ФАЙЛЫ ДЛЯ GOOGLE APPS SCRIPT
1) Code.gs — главный код сервера.
2) appsscript.json — необязательно, но желательно: явно задаёт права для Таблицы и Google Drive.

ЧТО СДЕЛАТЬ ПОШАГОВО

ШАГ 1. Создайте Google Таблицу
Назовите её, например: ИУС 2.0 — работы учеников.

ШАГ 2. Откройте Apps Script
В Google Таблице нажмите:
Расширения → Apps Script

ШАГ 3. Вставьте Code.gs
Удалите весь старый код и вставьте содержимое файла Code.gs.
Нажмите Сохранить.

ШАГ 4. Добавьте appsscript.json, если можете
В Apps Script нажмите шестерёнку ⚙ Настройки проекта.
Включите: Показывать файл манифеста appsscript.json.
Откройте appsscript.json и вставьте содержимое файла appsscript.json.
Нажмите Сохранить.
Если не получается — можно пропустить, но тогда обязательно запустите authorizeIUS.

ШАГ 5. Дайте разрешения
Вверху выберите функцию setupIUS и нажмите ▶ Запустить.
Разрешите доступ.
Потом выберите функцию authorizeIUS и нажмите ▶ Запустить.
Разрешите доступ к Google Drive.

Если появится «Приложение не проверено»:
Дополнительно → Перейти к проекту → Разрешить.

ШАГ 6. Проверьте диагностику
Выберите функцию diagnoseIUS и нажмите ▶ Запустить.
Должно быть сообщение: Таблица и DriveApp работают.

ШАГ 7. Разверните веб-приложение
Нажмите:
Развернуть → Новое развертывание → Тип: Веб-приложение

Настройки:
- Описание: ИУС 2.0
- Выполнять от имени: Меня
- У кого есть доступ: Все

Нажмите Развернуть.
Скопируйте ссылку, она должна заканчиваться на /exec.

ШАГ 8. Вставьте ссылку в apps-script-config.js
Откройте файл apps-script-config.js и вставьте ссылку:

window.APPS_SCRIPT_WEB_APP_URL = "https://script.google.com/macros/s/ВАША_ССЫЛКА/exec";

ШАГ 9. Загрузите файлы на GitHub
В репозиторий GitHub Pages загрузите/замените:
- index.html
- expert.html
- styles.css
- apps-script-config.js

Code.gs и appsscript.json на GitHub загружать не нужно.
Они нужны только в Apps Script.

ШАГ 10. Проверка
1) Откройте сайт ученика.
2) Зарегистрируйтесь как ученик.
3) Запишите короткий ответ.
4) Нажмите «Отправить работу учителю».
5) В Google Таблице должна появиться строка.
6) В Google Drive должна появиться папка «ИУС 2.0 — аудиозаписи учеников».
7) На expert.html нажмите «Показать отправленные работы».
8) Откройте работу и прослушайте аудио.

ВАЖНО
Если строка есть, но аудио пишет «Доступ запрещен: DriveApp», значит вы не запустили authorizeIUS или не сделали новую версию развертывания.

После любого изменения Code.gs обязательно:
Развернуть → Управление развертываниями → Изменить → Новая версия → Развернуть.


ИСПРАВЛЕНИЕ ВОСПРОИЗВЕДЕНИЯ АУДИО
Если работа открывается у учителя, но аудио не проигрывается, используйте этот комплект.
В нём expert.html не пытается проигрывать обычную ссылку Google Drive напрямую.
Он запрашивает у Apps Script действие action=audioData, получает аудио как data:audio/...;base64,... и вставляет его в тег audio.

После замены Code.gs обязательно сделайте:
1) Сохранить.
2) Запустить authorizeIUS.
3) Развернуть → Управление развертываниями → Изменить → Новая версия → Развернуть.
4) На GitHub заменить expert.html и нажать Ctrl+F5 на сайте.
